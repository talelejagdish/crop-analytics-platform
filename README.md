**2. Microservices with Domain-Driven Design (DDD)**

Each microservice owns its bounded context:
- **Crop Service**: Crop metadata, hierarchy definitions
- **Analysis Service**: Video processing, ML inference orchestration
- **User Service**: Authentication, authorization, user management
- **Notification Service**: Email, SMS, webhook notifications
- **Reporting Service**: Analytics, dashboards, exports

**Communication Patterns:**
- Synchronous: REST APIs for queries
- Asynchronous: Event-driven for commands
- Saga Pattern: Distributed transactions
